## Базовые сведения

### Базовый URL
```
http://localhost:8000/api
```

### Формат ответов
Все ответы возвращаются в формате JSON.

### Коды статусов
- `200 OK` - Успешный запрос
- `201 Created` - Ресурс успешно создан
- `204 No Content` - Успешное удаление
- `400 Bad Request` - Неверные параметры запроса
- `401 Unauthorized` - Требуется аутентификация
- `403 Forbidden` - Доступ запрещен
- `404 Not Found` - Ресурс не найден
- `500 Internal Server Error` - Внутренняя ошибка сервера

### Аутентификация
Большинство эндпоинтов требуют JWT токен в заголовке:
```
Authorization: Bearer <access_token>
```

## Аутентификация

### POST /api/auth/register
Регистрация нового пользователя. При регистрации автоматически создается дефолтное рабочее пространство.

**Аутентификация:** Не требуется

**Тело запроса:**
```json
{
  "email": "user@example.com",
  "password": "password123",
  "name": "Иван Иванов",
  "company_name": "ООО Компания"  // опционально
}
```

**Ответ (201 Created):**
```json
{
  "access_token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
  "refresh_token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
  "token_type": "bearer"
}
```

**Ошибки:**
- `400` - Пользователь с таким email уже существует

---

### POST /api/auth/login
Вход пользователя в систему.

**Аутентификация:** Не требуется

**Тело запроса:**
```json
{
  "email": "user@example.com",
  "password": "password123"
}
```

**Ответ (200 OK):**
```json
{
  "access_token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
  "refresh_token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
  "token_type": "bearer"
}
```

**Ошибки:**
- `401` - Неверный email или пароль
- `403` - Пользователь неактивен

## Чаты

### POST /api/chat/send
Отправка сообщения в чат и получение ответа от LLM.

**Аутентификация:** Требуется

**Тело запроса:**
```json
{
  "message": "Как составить бизнес-план?",
  "chat_id": 1,  // опционально, если не указан - создается новый чат
  "space_id": 1  // опционально, если не указан - используется дефолтное пространство
}
```

**Ответ (200 OK):**
```json
{
  "success": true,
  "chat_id": 1,
  "message_id": 42,
  "response": {
    "raw_text": "Бизнес-план состоит из следующих разделов...",
    "formatted_html": "<p>Бизнес-план состоит из следующих разделов...</p>",
    "timestamp": "2025-11-16T12:00:00",
    "category": "management",
    "probabilities": {
      "management": 0.85,
      "finance": 0.10,
      "general": 0.05
    }
  },
  "error": null
}
```

**Ошибки:**
- `400` - Сообщение не может быть пустым
- `404` - Пространство или чат не найдены

---

### GET /api/chat/history
Получение истории чатов пользователя.

**Аутентификация:** Требуется

**Query параметры:**
- `space_id` (int, опционально) - Фильтр по пространству
- `limit` (int, опционально, по умолчанию 50) - Количество чатов на странице (1-100)
- `offset` (int, опционально, по умолчанию 0) - Смещение для пагинации

**Пример запроса:**
```
GET /api/chat/history?space_id=1&limit=20&offset=0
```

**Ответ (200 OK):**
```json
{
  "chats": [
    {
      "id": 1,
      "title": "Как составить бизнес-план?",
      "space_id": 1,
      "space_name": "Моё рабочее пространство",
      "last_message": "Бизнес-план состоит из следующих разделов...",
      "last_message_at": "2025-11-16T12:00:00",
      "created_at": "2025-11-16T10:00:00",
      "updated_at": "2025-11-16T12:00:00"
    }
  ],
  "total": 1
}
```

### GET /api/chat/{chat_id}/messages
Получение сообщений в конкретном чате.

**Аутентификация:** Требуется

**Path параметры:**
- `chat_id` (int) - ID чата

**Query параметры:**
- `limit` (int, опционально, по умолчанию 100) - Количество сообщений (1-500)
- `offset` (int, опционально, по умолчанию 0) - Смещение для пагинации

**Пример запроса:**
```
GET /api/chat/1/messages?limit=50&offset=0
```

**Ответ (200 OK):**
```json
{
  "messages": [
    {
      "id": 1,
      "role": "user",
      "content": "Как составить бизнес-план?",
      "created_at": "2025-11-16T10:00:00"
    },
    {
      "id": 2,
      "role": "assistant",
      "content": "Бизнес-план состоит из следующих разделов...",
      "created_at": "2025-11-16T10:00:05"
    }
  ],
  "total": 2,
  "chat_id": 1,
  "chat_title": "Как составить бизнес-план?"
}
```

**Ошибки:**
- `404` - Чат не найден или нет доступа

- `space_id` (int, опционально) - Фильтр по пространству
- `limit` (int, опционально, по умолчанию 20) - Количество результатов на каждый тип (1-100)

**Примеры запросов:**
```
GET /api/search?q=маркетинг
GET /api/search?q=бюджет&type=notes
GET /api/search?q=стратегия&space_id=1&limit=10
```

## Примеры использования

### Пример: Полный цикл работы с API

```bash
# 1. Регистрация
curl -X POST "http://localhost:8000/api/auth/register" \
  -H "Content-Type: application/json" \
  -d '{
    "email": "user@example.com",
    "password": "password123",
    "name": "Иван Иванов"
  }'

# 2. Вход (получаем токен)
curl -X POST "http://localhost:8000/api/auth/login" \
  -H "Content-Type: application/json" \
  -d '{
    "email": "user@example.com",
    "password": "password123"
  }'

# 3. Создание пространства
curl -X POST "http://localhost:8000/api/spaces" \
  -H "Authorization: Bearer YOUR_ACCESS_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Мой проект",
    "description": "Описание проекта"
  }'

# 4. Создание заметки
curl -X POST "http://localhost:8000/api/notes/create" \
  -H "Authorization: Bearer YOUR_ACCESS_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "title": "Моя заметка",
    "content": "Содержимое заметки",
    "space_id": 1
  }'

# 5. Отправка сообщения в чат
curl -X POST "http://localhost:8000/api/chat/send" \
  -H "Authorization: Bearer YOUR_ACCESS_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "message": "Как составить бизнес-план?",
    "space_id": 1
  }'

# 6. Поиск
curl -X GET "http://localhost:8000/api/search?q=бизнес&type=all" \
  -H "Authorization: Bearer YOUR_ACCESS_TOKEN"
```

## Примечания

1. **JWT Токены**: Access токены имеют ограниченное время жизни. Используйте refresh токен для обновления.

2. **Дефолтное пространство**: При регистрации автоматически создается рабочее пространство "Моё рабочее пространство".

3. **Пагинация**: Все списки поддерживают пагинацию через параметры `limit` и `offset`.

4. **Фильтрация**: Многие эндпоинты поддерживают фильтрацию по пространству через параметр `space_id`.

5. **Права доступа**: Пользователь может работать только со своими данными. Попытки доступа к чужим ресурсам возвращают 404.

6. **Формат дат**: Все даты возвращаются в формате ISO 8601 (например, `2025-11-16T10:00:00`).
